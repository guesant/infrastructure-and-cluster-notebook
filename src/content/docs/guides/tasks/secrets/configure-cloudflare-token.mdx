> **Pré-requisitos:** domínio gerenciado pela Cloudflare, [cert-manager instalado](../../certificates/install-cert-manager/).
> **Versões testadas:** cert-manager v1.21.0.

O desafio DNS-01 usado em [criar um ClusterIssuer ACME](../../certificates/create-acme-clusterissuer/) precisa de uma credencial com permissão de escrita na zona DNS. Esta página cobre a criação de um token da Cloudflare com escopo mínimo, não a credencial completa da conta.

## Criar um token com escopo mínimo

Na Cloudflare, crie um token de API (não a Global API Key) com a permissão `Zone.DNS.Edit` (no seletor de permissões da interface, isso corresponde ao caminho `Zone` → `DNS` → `Edit`), com o escopo de recurso restrito à zona específica usada pelo cluster, não "All zones".

Um token com escopo mínimo limita o impacto de uma exposição acidental: se vazado, só permite editar registros DNS de uma zona específica, não gerenciar a conta inteira.

## Armazenar o token como Secret

Use a estratégia de segredos já escolhida para o ambiente (veja [visão geral de gerenciamento de segredos](../../../../learn/secrets-management/overview/)) em vez de aplicar diretamente sem versionamento, quando possível:

> **Executar em:** qualquer máquina com `KUBECONFIG` e acesso administrativo à API.

```bash
read -r -s -p "Token da API Cloudflare: " CLOUDFLARE_API_TOKEN
printf '\n'

kubectl --namespace cert-manager create secret generic cloudflare-api-token \
  --from-literal=api-token="${CLOUDFLARE_API_TOKEN}"
```

## Referenciar no ClusterIssuer

Ajuste o `solvers` do `ClusterIssuer` (veja [criar um ClusterIssuer ACME](../../certificates/create-acme-clusterissuer/)) para o provedor `cloudflare`:

```yaml
solvers:
  - dns01:
      cloudflare:
        apiTokenSecretRef:
          name: cloudflare-api-token
          key: api-token
```

## Validação

Emita um certificado de teste e confirme que o desafio DNS-01 completa; veja [diagnosticar uma emissão](../../certificates/install-cert-manager/#diagnosticar-uma-emissão).

## Troubleshooting

Se o `Challenge` falhar com erro de permissão, confirme que o token tem escopo `Zone.DNS.Edit` na zona correta: um token com apenas leitura (`Zone.DNS.Read`) falha silenciosamente ao tentar criar o registro TXT do desafio.

## Rollback

Revogue o token diretamente no painel da Cloudflare e remova o Secret:

```bash
kubectl --namespace cert-manager delete secret cloudflare-api-token
```
