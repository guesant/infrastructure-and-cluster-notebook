> **Pré-requisitos:** `sops` e `age` instalados na estação administrativa.
> **Versões testadas:** SOPS 3.9, age 1.2.

SOPS criptografa valores individuais de um arquivo YAML/JSON usando uma chave assimétrica, neste caso gerada pelo `age`. Veja [SOPS ou Sealed Secrets](../../../../learn/secrets-management/sops-vs-sealed-secrets/) antes de escolher esta estratégia.

## Gerar o par de chaves age

> **Executar em:** estação administrativa.

```bash
age-keygen -o age-key.txt
cat age-key.txt
```

A chave privada (`AGE-SECRET-KEY-...`) nunca deve ser commitada; veja [proteger chaves age](../../../../operations/backups/protect-age-keys/) para onde guardá-la. A chave pública (`age1...`, no comentário do arquivo) é o que vai na configuração do SOPS.

## Configurar o `.sops.yaml`

Na raiz do repositório GitOps:

```bash
read -r -p "Chave pública age (age1...): " AGE_PUBLIC_KEY

cat >.sops.yaml <<EOF
creation_rules:
  - path_regex: gitops/apps/security/secrets/.*\.yaml$
    age: ${AGE_PUBLIC_KEY}
EOF
```

Ajuste `path_regex` ao diretório real onde os arquivos de segredo cifrados ficarão: o SOPS aplica a regra da primeira `path_regex` que corresponder ao caminho do arquivo.

## Criptografar um arquivo

```bash
read -r -p "Caminho do Secret em claro (será sobrescrito, cifrado): " SECRET_FILE_PATH

export SOPS_AGE_KEY_FILE=age-key.txt
sops --encrypt --in-place "${SECRET_FILE_PATH}"
```

## Validação

```bash
sops --decrypt "${SECRET_FILE_PATH}"
```

Deve exibir o conteúdo original. Confirme também que o arquivo commitado no Git mostra os valores como texto cifrado (`ENC[AES256_GCM,...]`), não em claro.

## Troubleshooting

Se `sops --decrypt` falhar com "no key found", confirme que `SOPS_AGE_KEY_FILE` aponta para o arquivo correto ou que a chave está em `~/.config/sops/age/keys.txt` (local padrão).

## Rollback

Reverter para texto claro (`sops --decrypt --in-place`) só deve acontecer temporariamente para edição, nunca como estado a ser commitado.
