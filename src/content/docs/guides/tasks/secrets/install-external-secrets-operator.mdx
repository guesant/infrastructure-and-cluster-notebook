> **Pré-requisitos:** kubeconfig com acesso administrativo à API, um backend externo já disponível (OpenBao, Vault, ou provedor de nuvem).
> **Versões testadas:** External Secrets Operator 0.19.

Veja [External Secrets Operator](../../../../learn/secrets-management/external-secrets/) para o conceito antes de instalar. Esta página cobre a instalação e um `ExternalSecret` de exemplo apontando para um OpenBao/Vault já disponível.

## Instalar o operator

> **Executar em:** qualquer máquina com `KUBECONFIG`, Helm e acesso administrativo à API.

```bash
read -r -p "Versão do chart external-secrets: " ESO_VERSION

helm upgrade --install external-secrets external-secrets \
  --repo https://charts.external-secrets.io \
  --version "${ESO_VERSION}" \
  --namespace external-secrets \
  --create-namespace \
  --set installCRDs=true
```

## Configurar o SecretStore

```bash
read -r -p "Endereço do backend (ex.: https://openbao.exemplo:8200): " BACKEND_ADDRESS
read -r -p "Caminho do secrets engine KV: " KV_PATH

kubectl apply -f - <<EOF
apiVersion: external-secrets.io/v1beta1
kind: SecretStore
metadata:
  name: openbao-backend
  namespace: default
spec:
  provider:
    vault:
      server: "${BACKEND_ADDRESS}"
      path: "${KV_PATH}"
      version: v2
      auth:
        kubernetes:
          mountPath: kubernetes
          role: external-secrets
EOF
```

O método `kubernetes` de autenticação usa a identidade da ServiceAccount do operator para autenticar no backend, evitando uma credencial estática adicional; veja [o problema do bootstrap](../../../../learn/secrets-management/bootstrap-problem/).

## Criar um ExternalSecret

```bash
read -r -p "Namespace de destino: " EXTERNAL_SECRET_NAMESPACE
read -r -p "Nome do Secret Kubernetes a gerar: " EXTERNAL_SECRET_NAME
read -r -p "Caminho remoto da credencial no backend: " REMOTE_KEY

kubectl apply -f - <<EOF
apiVersion: external-secrets.io/v1beta1
kind: ExternalSecret
metadata:
  name: ${EXTERNAL_SECRET_NAME}
  namespace: ${EXTERNAL_SECRET_NAMESPACE}
spec:
  secretStoreRef:
    name: openbao-backend
    kind: SecretStore
  target:
    name: ${EXTERNAL_SECRET_NAME}
  data:
    - secretKey: value
      remoteRef:
        key: ${REMOTE_KEY}
EOF
```

## Validação

> **Executar em:** qualquer máquina com `KUBECONFIG` e acesso à API.

```bash
kubectl --namespace "${EXTERNAL_SECRET_NAMESPACE}" get externalsecret "${EXTERNAL_SECRET_NAME}"
kubectl --namespace "${EXTERNAL_SECRET_NAMESPACE}" get secret "${EXTERNAL_SECRET_NAME}"
```

O `ExternalSecret` deve mostrar `STATUS: SecretSynced`, e o `Secret` correspondente deve existir.

## Troubleshooting

Se `STATUS` mostrar erro de autenticação, confirme o `role` configurado no backend e se a ServiceAccount usada tem a role correta associada: o método `kubernetes` depende de TokenReview funcionando entre o cluster e o backend.

## Rollback

```bash
kubectl --namespace "${EXTERNAL_SECRET_NAMESPACE}" delete externalsecret "${EXTERNAL_SECRET_NAME}"
```
