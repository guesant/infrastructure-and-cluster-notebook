> **Pré-requisitos:** [SOPS configurado com age](../configure-sops-with-age/), Argo CD instalado.
> **Versões testadas:** Argo CD (chart 10.1.3), SOPS 3.9.

O Argo CD não decodifica arquivos SOPS nativamente: é necessário um plugin CMP (Config Management Plugin) que decifra os arquivos antes de gerar os manifests finais.

## Disponibilizar a chave privada ao Argo CD

A chave age precisa estar acessível ao plugin, como um Secret no namespace `argocd`:

> **Executar em:** qualquer máquina com `KUBECONFIG` e acesso administrativo à API.

```bash
kubectl --namespace argocd create secret generic sops-age-key \
  --from-file=key.txt=age-key.txt
```

Esse Secret não deve ser gerenciado pelo próprio GitOps: é parte do bootstrap manual, como o token do K3s.

## Configurar o CMP como sidecar do repo-server

Edite o Deployment `argocd-repo-server` para adicionar um sidecar CMP (via `kustomize`, Helm values, ou patch direto):

```yaml
containers:
  - name: sops-cmp
    image: viaductoss/argocd-sops:latest
    command: [/var/run/argocd/argocd-cmp-server]
    volumeMounts:
      - name: sops-age-key
        mountPath: /age-keys
        readOnly: true
      - name: cmp-tmp
        mountPath: /tmp
    env:
      - name: SOPS_AGE_KEY_FILE
        value: /age-keys/key.txt
volumes:
  - name: sops-age-key
    secret:
      secretName: sops-age-key
  - name: cmp-tmp
    emptyDir: {}
```

Fixe a versão da imagem do plugin em vez de `latest` em um ambiente real: o exemplo usa `latest` apenas ilustrativamente.

## Referenciar o plugin em uma Application

```yaml
apiVersion: argoproj.io/v1alpha1
kind: Application
spec:
  source:
    plugin:
      name: sops-cmp
```

## Validação

> **Executar em:** qualquer máquina com `KUBECONFIG` e acesso à API.

```bash
kubectl --namespace argocd logs deployment/argocd-repo-server -c sops-cmp --tail=50
argocd app get <nome-da-application>
```

O `diff` da Application deve mostrar o `Secret` decodificado corretamente, não o conteúdo cifrado.

## Troubleshooting

Se a Application falhar ao gerar manifests, confirme que o Secret `sops-age-key` existe no namespace `argocd` e que a versão da imagem do plugin CMP é compatível com a versão do Argo CD instalada.

## Rollback

Remova o `plugin` da `Application` e o sidecar do `argocd-repo-server` para voltar à renderização padrão: arquivos cifrados voltarão a aparecer como texto cifrado nos manifests aplicados, o que provavelmente quebra a aplicação até a reversão completa.
