> **Pré-requisitos:** estratégia de segredos escolhida (veja [visão geral](../../../../learn/secrets-management/overview/)).
> **Versões testadas:** não aplicável; esta página é um roteiro que aponta para as páginas de instalação de cada estratégia.

Esta página funciona como roteiro para o bootstrap de qualquer uma das estratégias de segredos deste notebook: o passo manual único que precisa acontecer fora do GitOps antes que a sincronização automatizada comece. Veja [o problema do bootstrap](../../../../learn/secrets-management/bootstrap-problem/) para o conceito.

## Roteiro por estratégia

| Estratégia | Página de instalação | O que é aplicado manualmente |
| --- | --- | --- |
| SOPS + age | [configurar SOPS com age](../configure-sops-with-age/) | Nada no cluster; a chave privada fica na estação administrativa/CI. |
| Sealed Secrets | [instalar o Sealed Secrets](../install-sealed-secrets/) | O controller gera seu próprio par de chaves na primeira execução. |
| External Secrets Operator | [instalar o External Secrets Operator](../install-external-secrets-operator/) | A credencial ou role de autenticação no `SecretStore`. |
| OpenBao | [instalar o OpenBao](../install-openbao/) | As chaves de unseal, capturadas na inicialização. |
| Infisical | [instalar o Infisical](../install-infisical/) | O Secret com `clientId`/`clientSecret` da Machine Identity. |

## Checklist geral de bootstrap

- [ ] O material de bootstrap (chave, credencial, chaves de unseal) foi gerado e capturado no momento certo: muitos backends só exibem esse material uma única vez.
- [ ] O material foi guardado em um gerenciador de segredos ou cofre físico, nunca em texto claro no repositório GitOps.
- [ ] O procedimento de aplicação manual está documentado (esta página, ou uma versão específica do ambiente) e não depende da memória de quem o executou.
- [ ] O restante da configuração (conexão, autenticação, sincronização) está versionado no Git normalmente, a partir deste ponto.
- [ ] O procedimento de recuperação em caso de perda do material de bootstrap está documentado; veja [recuperar o gerenciamento de segredos](../../../../operations/disaster-recovery/recover-secret-management/).
