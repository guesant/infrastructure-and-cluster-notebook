> **Para quem é:** quem já tem 1 servidor rodando e quer adicionar servidor-1, servidor-2, etc.
> **Pré-requisito:** primeiro servidor em [Inicializar o primeiro servidor](../first-server/).

Adicionar um novo servidor a um cluster K3s multinode é similar ao single-node, mas com etapa extra: registrar o novo servidor como parte do quorum de etcd. Este guia cobre os passos específicos.

## Antes de começar

- Tenha o **token de join** do primeiro servidor (do arquivo `/var/lib/rancher/k3s/server/node-token` no primeiro servidor)
- Tenha o **IP ou DNS** do primeiro servidor (ex.: `10.0.0.10`)
- Novo servidor já tem Debian/Ubuntu + firewall configurado conforme [requisitos de rede](../network-requirements/)

## Passo 1: Instalar K3s no novo servidor

No novo servidor (servidor-1), execute:

```bash
curl -sfL https://get.k3s.io | INSTALL_K3S_VERSION=v1.36.0+k3s1 sh -s - server \
  --server https://<ip-primeiro-servidor>:6443 \
  --token <seu-token-de-join> \
  --write-kubeconfig-mode 644
```

Substitua:
- `<ip-primeiro-servidor>`: IP ou DNS do servidor-0 (ex.: `10.0.0.10`)
- `<seu-token-de-join>`: conteúdo de `/var/lib/rancher/k3s/server/node-token` do servidor-0

Exemplo real:

```bash
curl -sfL https://get.k3s.io | INSTALL_K3S_VERSION=v1.36.0+k3s1 sh -s - server \
  --server https://10.0.0.10:6443 \
  --token 'K102abc123::server:xyz789' \
  --write-kubeconfig-mode 644
```

## Passo 2: Aguardar sincronização

Espere de 20 a 30 segundos enquanto o etcd sincroniza entre os dois servidores. No servidor-1,
verifique:

```bash
sudo kubectl get nodes
```

Ambos os servidores devem aparecer como `Ready`:

```text
NAME         STATUS   ROLES                  AGE   VERSION
servidor-0   Ready    control-plane,master   2m    v1.36.0+k3s1
servidor-1   Ready    control-plane,master   30s   v1.36.0+k3s1
```

## Passo 3: Adicionar servidor-2 (se usando 3 servidores)

Repita os passos 1 e 2 no servidor-2, apontando ainda para servidor-0 como `--server`:

```bash
curl -sfL https://get.k3s.io | INSTALL_K3S_VERSION=v1.36.0+k3s1 sh -s - server \
  --server https://10.0.0.10:6443 \
  --token 'K102abc123::server:xyz789' \
  --write-kubeconfig-mode 644
```

Quando servidor-2 estiver `Ready`, o cluster tem quorum completo: 3 servidores, quorum = 2.

## Verificação de quorum

Depois que os 3 servidores estão `Ready`, o sinal mais confiável de que o etcd tem quorum
saudável é justamente esse: todos os servidores aparecendo como `Ready` simultaneamente, já que um
servidor sem conexão ao quorum não consegue manter esse status por muito tempo. Para investigar
mais a fundo em caso de dúvida, os logs do próprio K3s expõem o estado do etcd:

```bash
sudo journalctl -u k3s -f | grep etcd
```

Nenhuma mensagem de erro relacionada a Raft nos logs recentes indica que os membros estão
sincronizados.

## Problemas comuns

### "Connection refused" ou "timeout" ao adicionar servidor

**Causa:** firewall bloqueando porta 6443 entre servidores.

**Solução:**

```bash
# No servidor novo, testar conectividade ao primeiro
sudo nc -zv 10.0.0.10 6443
```

Se falhar, configure o firewall conforme [requisitos de rede](../network-requirements/).

### Servidor novo fica "NotReady"

**Causa:** etcd não sincronizou ainda, ou há problema de disco ou memória.

**Solução:**

```bash
# Verificar logs
sudo journalctl -u k3s -n 50

# Aguardar 1-2 minutos e tentar novamente
sleep 60 && sudo kubectl get nodes
```

### "token does not have correct format"

**Causa:** token copiado incorretamente (espaços, quebras de linha).

**Solução:** copie novamente do arquivo no servidor-0 com cuidado:

```bash
# No servidor-0
sudo cat /var/lib/rancher/k3s/server/node-token
# copiar sem espaços/quebras
```
